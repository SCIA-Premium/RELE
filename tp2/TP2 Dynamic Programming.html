I. Chaîne de Markov finie¶

Un processus de dcision markovien est bas sur l'hypothse de Markov, qui stipule qu'en connaissant l'tat l'instant $t$, on sait tout ce qui s'est pass auparavant.

Mathmatiquement, cette hypothse signifie que la probabilit d'une variable $X_{t+1}$ sachant l'tat $ X_t $ est indpendante des tats prcdents :

$$ \mathbb{P}(X_{t+1}| X_{t},X_{t-1},\dots) = \mathbb{P}(X_{t+1} | X_{t}) $$

Q4. A l'aide des thormes des probabilits, dmontrez le rsultat suivant :

$$ \mathbb {P} \{X_{t+1} = y \} = \sum_{x \in S} \mathbb {P} \{ X_{t+1} = y \, | \, X_t = x \} \cdot \mathbb {P} \{ X_t = x \} $$
$$ \begin{align} \sum_{x \in S} \mathbb {P} \{ X_{t+1} = y \, | \, X_t = x \} \cdot \mathbb {P} \{ X_t = x \} &= \sum_{x \in S} \frac{P \{ X_{t+1} = y \} \cdot P \{ X_t = x \, | \, X_{t+1} = y \}}{P \{ X_{t} = x \}} \cdot P \{ X_t = x \} \\ &= P \{ X_{t+1} = y \} \sum_{x \in S} \frac{ P \{ X_t = x \}}{P \{ X_{t} = x \}} \cdot P \{ X_t = x \} \text{ car } P\{X^{}_{t+1} = y\} \text{ ne dpend pas de x} \\ &= P \{ X_{t+1} = y \} \sum_{x \in S} P \{ X_t = x \} \\ &= P \{ X_{t+1} = y \} \text{ car la somme des } P \{ X_t = x \} \text{ avec } x \in S \text{ est gale 1} \\ \end{align} $$

Autrement dit, pour connatre la probabilit d'avoir $y$ demain, il faut considrer tous les cas aujourd'hui ! Avec la notation matricielle, cela s'crit :

$$ \mathbb {P} \{X_{t+1} = x_j \} = \sum_{x_i \in S} \mathbb {P} \{ X_{t+1} = x_j | X_t = x_i \} \cdot \mathbb {P} \{ X_t = x_i \} = \sum_{i \in \{1,\dots,n\}} p_{ij} \cdot \psi_{t,i} = \pmb{\Psi}_{t} \cdot \pmb{P} $$

avec $ \psi_t $ , la distribution de probabilit l'instant $t$.

En appliquant cette formule rcursivement, on obtient la probabilit de la mto au N-me jour partir d'un simple calcul matriciel :

$$ \pmb{\Psi}_{t+N} = \pmb{\Psi}_{t+N-1} \cdot \pmb{P} = \cdots = \pmb{\Psi}_{t} \cdot \pmb{P}^N $$

II. Convergence vers un état stationnaire d'une chaîne de Markov.¶

Cette partie illustre la convergence d'une chane de Markov vers une distribution stationnaire.

Une distribution $ \psi^\star $ est dite stationnaire ou invariante si $$\pmb{\Psi}^\star = \pmb{\Psi}^\star \cdot \pmb{P}$$

Ainsi, $$\pmb{\Psi}^\star = \pmb{\Psi}^\star \cdot\pmb{P}^k \text{ pour tout } k.$$

De plus, si $ X_0 $ a la distribution $ \pmb{\Psi}^\star $, alors $ X_t $ aura la mme distribution pour tout $ t $.

*Thorme : toute matrice stochastique $\pmb{P}$ admet au moins une distribution stationnaire $\pmb{\Psi}$*

III. 1. Estimation de la fonction de valeur d'un gridword¶

Nous avons vu en cours que :

$$v_\pi (s) = \mathbb{E}_\pi \left( G_t | s \right) = \sum_{s'} p(s'|s, a)\left[r+\gamma v_\pi(s') \right]$$

Dans le cas o les dynamiques de l'environnement sont entirement connus, $p(s'|s, a)$ peut s'exprimer sous la forme d'un tensor et l'quation prcdente aboutit un systme d'quations linaires. Le problme est donc rsolvable, mais la rsolution risque d'tre longue si l'environnement est grand.

On cherche plutt une rsolution itrative en appliquant les principes de la programmation dynamique. Concrtement, on part d'une fonction de valeur arbitraire $v_0$ (par exemple nulle partout), puis on y applique chaque tape l'quation de Bellman : $$v_{k+1} (s) = \sum_{s'} p(s'|s, a)\left[r+\gamma v_k(s') \right]$$ Lorsque l'algorithme a converg vers un point fixe $v_\infty$, nous avons fini d'valuer $v_\pi$, puisque ce dernier est l'unique point fixe de la fonction de valeur.

Cet algorithme est appel l'valuation itrative de la politique.

from tabulate import tabulate
import typing as t
from dataclasses import dataclass, field
import random
import torch

Action = t.Literal["L", "R", "U" , "D"]
CUBE_SIDE = 6

@dataclass
class State: 
    """
    It represents any cell in the world
    """
    cell: int
    value: int = 0
    
    def __post_init__(self):
        self.bounds = {
            'L': self.cell - self.cell % CUBE_SIDE,
            'R': self.cell - self.cell % CUBE_SIDE + (CUBE_SIDE - 1),
            'U': self.cell % CUBE_SIDE,
            'D': self.cell % CUBE_SIDE + CUBE_SIDE * (CUBE_SIDE - 1),
        }
        self.neighbors = [self.act(a) for a in "LRUD"]
        assert all(i >= 0 and i < CUBE_SIDE*CUBE_SIDE for i in self.neighbors)
    
    def is_termination(self):
        return self.cell in {0, CUBE_SIDE * CUBE_SIDE - 1}

    def act(self, a: Action):
        """
        Get next state
        """
        if a == 'L': 
            return min(self.bounds['R'], max(self.bounds['L'], self.cell - 1))
        if a == 'R': 
            return min(self.bounds['R'], max(self.bounds['L'], self.cell + 1))
        if a == 'U': 
            return min(self.bounds['D'], max(self.bounds['U'], self.cell - 4))
        if a == 'D':
            return min(self.bounds['D'], max(self.bounds['U'], self.cell + 4))
        raise ValueError('Unexpected action')
    

def init_states():
    return [State(i) for i in range(CUBE_SIDE * CUBE_SIDE)]


@dataclass
class Env:
    states: t.List[State] = field(default_factory=init_states)

La politique gloutonne cherche uniquement exploiter, sans aucune exploration. A chaque instant, elle choisit l'action qui permet de maximiser la fonction de valeur :

$$\pi(s) = \text{argmax}_a \sum_{s'} p(s'|s,a)[r+\gamma V(s')]$$

III. 4. Algorithme value iteration¶

Une autre variante conserve la politique alatoire tout en long de l'entranement, mais met jour la fonction de valeur avec l'quation suivante :

$$v_{k+1} (s) = \max_{a} \sum_{s'} p(s'|s, a)\left[r+\gamma v_k(s') \right]$$

Une fois que la fonction de valeur a converg, on calcule la politique avec :

$$\pi(s) = argmax_a \sum_{s'} p(s'|s,a)[r+\gamma V(s')]$$
