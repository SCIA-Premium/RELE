L'environnement Taxi est un parking avec plusieurs lments. Il y a quatre emplacements dsigns dans le monde de la grille indiqus par R (ed), G (reen), Y (ellow) et B (lue). Lorsque l'pisode commence, le taxi dmarre sur une case alatoire et le passager se trouve un endroit alatoire. Le taxi se rend l'emplacement du passager, prend le passager, le conduit jusqu' sa destination (un autre des quatre emplacements spcifis), puis dpose le passager. Une fois le passager dpos, l'pisode se termine.

Il existe 6 actions dterministes discrtes :

  • 0 : se dplacer vers le sud
  • 1 : se dplacer vers le nord
  • 2 : se dplacer vers l'est
  • 3 : se dplacer vers l'ouest
  • 4 : prendre un passager
  • 5 : dposer un passager

L'output de env.step est un Tuple[ObsType, float, bool, bool, dict], il contient :

  • observation (objet) - ce sera un lment de l'espace d'observation de l'environnement. Cela peut, par exemple, tre un tableau numpy contenant les positions et les vitesses de certains objets.
  • reward (float) - Le montant de la rcompense retourne la suite de l'action.
  • terminated (bool) si un tat terminal est atteint. Dans ce cas, d'autres appels step() pourraient renvoyer des rsultats indfinis.
  • truncated (bool) Gnralement une limite de temps, mais peut galement tre utilise pour indiquer que l'agent sort physiquement des limites. Peut tre utilis pour mettre fin l'pisode prmaturment avant qu'un tat terminal ne soit atteint.
  • info (dict) - info contient des informations de diagnostic auxiliaires (utiles pour le dbogage, l'apprentissage et la journalisation). Cela peut, par exemple, contenir : des mtriques qui dcrivent l'tat de performance de l'agent, des variables qui sont masques dans les observations ou des termes de rcompense individuels qui sont combins pour produire la rcompense totale. Il peut galement contenir des informations qui distinguent la troncature et la terminaison, mais cela est dconseill en faveur du retour de deux boolens, et sera supprim dans une future version.

def play_and_train(env: gym.core.Env, agent: QLearningAgent, t_max=int(1e4)) -> float:
    """
    This function should 
    - run a full game, actions given by agent.getAction(s)
    - train agent using agent.update(...) whenever possible
    - return total rewardb
    """
    total_reward = 0.0
    s, info = env.reset()
    
    for t in range(t_max):
        # TODO get agent to pick action given state s
        a = agent.get_action(s)
        next_s, r, done, truncated, info = env.step(a)

        # TODO train agent for state s
        if not done:
            agent.update(s, a, r, next_s)
        
        s = next_s
        total_reward += r
        if done or truncated:
            break
        
    return total_reward

4. Experience replay

Les algorithmes off-policy peuvent s'entraner grce des trajectoires anciennement obtenues. Tirons parti de cette proprit pour amliorer l'efficacit de nos algorithmes et les permettre de converger plus rapidement.

L'ide gnrale est de collecter les tuplets <s,a,r,s'> dans un buffer, puis de mettre jour la fonction Q sur l'ensemble de ces tuplets. Plus en dtails, voici l'algorithme suivre

S'entraner avec un experience replay

  1. Echantillonner un tuplet <s,a,r,s'>.
  2. Stocker ce tuplet dans un buffer FIFO : si le buffer est plein, on supprimer les donnes arrives en premier.
  3. Choisir alatoirement K tuplets du buffer et mettre jour la fonction Q sur ces tuplets.

class ReplayBuffer(object):
    def __init__(self, size):
        """
        Create Replay buffer.
        Parameters
        ----------
        size: int
            Max number of transitions to store in the buffer. When the buffer
            overflows the old memories are dropped.
            
        Note: for this assignment you can pick any data structure you want.
              If you want to keep it simple, you can store a list of tuples of (s, a, r, s') in self._storage
              However you may find out there are faster and/or more memory-efficient ways to do so.
        """
        self._maxsize = size
        self._next_idx = 0
        self._storage = []
        self._n_transitions_stored = 0

    def __len__(self):
        return self._n_transitions_stored
    
    def add(self, s, a, r, next_s):
        """
        Add a new experience to memory.
        Parameters
        ----------
        s: np.ndarray
            Current state
        a: int
            Chosen action
        r: float
            Received reward
        next_s: np.ndarray
            Next state
        """
        if self._n_transitions_stored < self._maxsize:
            self._storage.append((s, a, r, next_s))
            self._n_transitions_stored += 1
        else:
            self._storage[self._next_idx] = (s, a, r, next_s)
        self._next_idx = (self._next_idx + 1) % self._maxsize

    def sample(self, batch_size):
        """
        Sample a batch of experiences.
        Parameters
        ----------
        batch_size: int
            How many transitions to sample.
        Returns
        -------
        states: np.ndarray
            array of states
        actions: np.ndarray
            array of actions executed given state
        rewards: np.ndarray
            rewards received for executing given actions
        next_states: np.ndarray
            next state yielded by executing given actions
        """
        # TODO sample batch_size many elements from self._storage
        idx = np.random.choice(len(self._storage), batch_size)
        return [self._storage[i] for i in idx]
    
    def clear(self):
        self._storage = []
        self._n_transitions_stored = 0

def play_and_train_replay(
    env: gym.core.Env, 
    agent: QLearningAgentEpsScheduling, 
    t_max: int = int(1e4),

    This function should 
    - run a full game, actions given by agent.getAction(s)
    - train agent using agent.update(...) whenever possible
    - return total rewardb
